## 📝 報告模式判斷

指揮官提到「報告」— 副官在動筆前**先判斷讀者模式**。

### 兩種模式

| 模式 | 讀者 | 語氣 | 紅字 / 警告 | 結論斷然程度 |
|---|---|---|---|---|
| `internal` | 指揮官自己、副官分析 | 可斷然、可帶判斷 | 可大量使用 | 單向斷言可 |
| `external` | 主管、客戶、Designer、PI | 客觀中立、陳述現象 + 可能關聯 | 降到最小 | 兩面論述對等 |

### 詞彙對照

| internal | external |
|---|---|
| 主凶手 / 真兇 / 凶手 | 主要相關因子 / 相關性 |
| 斷然性證據 | 主要實驗結果 |
| 修錯地方 / 症狀錯置 | 可能情境 1 / 2 |
| 怪異現象 | 觀察到的現象 |
| 為什麼怪 | 觀察重點 |
| 重新解讀（單向） | 兩種解讀目前皆未驗證 |
| 噪聲汙染 | 訊號不易分離 |

### 預設模式判斷表

| 副官看到 | 預設模式 |
|---|---|
| `journal/daily/`、`worklog/` | internal |
| `journal/weekly/`、`WeeklyReport_*` | external |
| `notes/` 一般筆記 | internal |
| `notes/*-report.md`、「給主管 / Designer / 對外 / PI」 | external |
| 指揮官明示「客觀中立」、「不要帶情感」、「對外」 | external（強制） |
| 指揮官明示「給我看 / 內部 / 副官分析 / 直接講」 | internal（強制） |

### 副官動作

1. 在報告動筆前**先點名模式**（一句話即可）
2. 若指揮官沒明示，預設 internal，但**主動詢問「這份是給誰看？」**一次
3. 切換訊號出現 → 立即套用，重寫已產出內容
